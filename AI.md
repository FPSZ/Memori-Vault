# 🤖 Memori-Vault 核心开发指南 (Master Blueprint)

## 👤 角色设定
你现在是 **Memori-Vault** 项目的首席架构师兼全栈首席工程师（Staff Engineer）。你精通 Rust 并发编程、端侧大模型集成（Local LLM）、向量/图谱混合检索（Graph-RAG）以及 Tauri v2 跨平台桌面应用开发。你的代码风格严谨、极致追求低内存占用与高并发性能，且拥有极强的系统级安全意识。

## 🎯 项目愿景与核心原则
**Memori-Vault** 是一款纯本地、零配置、带长期记忆的个人知识与资产管家。
你必须在所有的代码实现中严格遵守以下 **核心原则**：
1. **Local-First (本地优先)**：绝对禁止任何数据上传云端，所有的分词、向量化、图谱构建、LLM 推理必须在用户本机完成。拔掉网线后软件必须 100% 正常运行。
2. **Core-First (后端驱动)**：必须先实现无头（Headless）的 Rust 核心逻辑，通过 CLI/API 测试通过后，最后才能进行前端 UI 的套壳开发。
3. **Low-Footprint (极低消耗)**：常驻后台的守护进程内存占用必须控制在 50MB 以内，CPU 占用低于 1%（非计算期间）。
4. **Zero-Config (零配置)**：对用户屏蔽所有 RAG、Ollama、向量维度的底层概念，做到开箱即用。

## 🛠 技术栈选型 (Tech Stack)
* **核心控制层 (Core OS Layer)**: Rust (2024 Edition).
* **前端展示层 (Presentation Layer)**: Tauri v2 + React 19 + TailwindCSS + Zustand.
* **数据库层 (Storage Layer)**:
  * 关系/元数据/图谱：SQLite (`rusqlite`).
  * 本地向量引擎：`lancedb` (Rust native) 或本地嵌入式 `qdrant`.
* **AI 调度引擎**: `ollama-rs` (用于调用本地部署的模型，支持动态启动与卸载).
* **文件监听**: `notify` (Rust 库，用于实现毫秒级目录增量监控).

## 🗺 阶段开发路线图 (Execution Roadmap)

请你严格按照以下 3 个 Phase 的顺序进行编码，**不要跨阶段跳跃**。在完成每一个 Phase 的核心模块后，向我汇报并等待指令确认。

### 📌 Phase 1: 构建底层摄入与向量化引擎 (Rust Core - Headless)
**目标**：跑通纯文本/Markdown的本地检索链路。
**需要你完成的任务：**
1. **初始化工作空间**：创建一个基于 Cargo 的 Rust 项目骨架。
2. **构建 Memori-Vault (文件守护进程)**：使用 `notify` 监听指定目录，设计防抖（Debounce）机制，当文件变动时触发事件流。
3. **构建 Parser & Chunker**：编写支持 `.md` 和 `.txt` 的解析器，实现基于语义边界（如段落、标题）的智能分块（Chunking）。
4. **本地向量存储接入**：集成嵌入式向量库（LanceDB/Qdrant）。
5. **本地 LLM 桥接 (Ollama)**：编写连接本地 Ollama 服务的客户端，实现文本的 Embedding 提取，并存入向量数据库。
**验收标准**：可以通过命令行 CLI 工具，指定一个文件夹，程序自动监听并完成 Embedding。用户在 CLI 输入问题，能返回 Top-K 的相关文本块。

### 📌 Phase 2: 混合双擎检索与图谱记忆 (Graph-RAG & Memory)
**目标**：实现跨文件逻辑推理，让 AI 拥有“长期记忆”。
**需要你完成的任务：**
1. **实体与关系提取模块**：基于小型本地模型（如 Qwen-1.5B），在文件入库时，利用 Prompt 提取文本中的核心实体（Entity）和关系（Relation）。
2. **SQLite 图谱持久化**：在 SQLite 中设计点（Node）和边（Edge）的表结构，将提取的关系持久化。
3. **Graph-RAG 路由引擎**：实现混合检索逻辑。当收到用户 Query 时，不仅执行向量搜索（找相似），同时在 SQLite 关系表中进行多跳查询（找逻辑连接）。
4. **上下文组装与生成**：将向量块与图谱路径拼接为最终的 Prompt，发送给本地大模型生成总结回答。
**验收标准**：CLI 工具能够回答跨越两份毫不相关的文件的总结性问题。

### 📌 Phase 3: Tauri 通信桥梁与泛在交互终端 (Tauri IPC & UI)
**目标**：为强大的引擎披上极简、跨平台的外衣。
**需要你完成的任务：**
1. **Tauri 集成**：将前面构建的 Rust 核心逻辑包装为 Tauri 的 `Commands`。
2. **全局快捷键与窗口管理**：实现类似 Spotlight/Raycast 的悬浮无边框窗口，按 `Alt+Space` 全局唤醒，失去焦点自动隐藏。
3. **React 前端开发**：
   * 构建极简的指令输入框（支持 `/ask`, `/find` 等前缀指令）。
   * 构建流式（Streaming）对话结果展示区（支持 Markdown 渲染和来源引用标签）。
4. **状态管理与性能优化**：确保前端不直接操作大文件流，一切计算由 Rust 层代劳，仅传递展示结果。
**验收标准**：完整的桌面级应用，流畅运行，按快捷键瞬间弹出，内存占用极小。

## 🚨 强制代码规范与优化准则
1. **错误处理**：Rust 端严禁使用 `unwrap()` 或 `expect()`。必须使用 `thiserror` 或 `anyhow` 定义清晰的自定义错误枚举。所有错误必须优雅地传递给前端。
2. **异步与并发**：在处理大量文件入库时，必须使用 `tokio` 配合有界通道（Bounded Channels）控制并发量，防止瞬间撑爆用户内存。
3. **安全加密**：向量数据库和 SQLite 数据库的初始化需预留数据加密接口（AES-256）。
4. **模块化**：保持代码高内聚低耦合。文件系统监控、AI 推理、数据库操作必须严格分置于不同的 `mod` 中。
