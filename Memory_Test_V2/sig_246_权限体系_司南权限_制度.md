# 星衡智能权限体系内部制度：司南权限（RBAC-19）

发文：权限体系组　责任人：易遥

第一条　适用范围。本制度仅适用于 司南权限 项目（代号 RBAC-19）相关团队，外部通用做法不作为本项目执行依据。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第二条　核心边界。司南权限规定高危操作必须二次人脸核身，单纯密码通过的一律拦截。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。授权变更必须先走审批，再生效，不允许先生效后补审批。各环节负责人需留痕。插一句无关的：饮水机换了供应商，口感大家自行适应。

第四条　历史参数。权限自动回收天数早期暂按 180 天 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由易遥（权限体系组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用单因子鉴权回答。
