# 星衡智能硬件协同内部制度：磐石终端（HW-55）

发文：硬件协同组　责任人：聂川

第一条　适用范围。本制度仅适用于 磐石终端 项目（代号 HW-55）相关团队，外部通用做法不作为本项目执行依据。群里有人问能不能远程，这块按现有制度走，不在本议题。

第二条　核心边界。磐石终端规定固件灰度必须先 50 台工程机跑满 48 小时再放量。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。固件回滚必须先停 OTA，再下发回滚包，最后才解封下载。各环节负责人需留痕。插一句无关的：饮水机换了供应商，口感大家自行适应。

第四条　历史参数。工程机验证时长早期暂按 24 小时 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由聂川（硬件协同组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用手机 500 次电池循环回答。
