# 星衡智能安全合规内部制度：玄武密钥（SEC-41）

发文：安全办公室　责任人：周曼青

第一条　适用范围。本制度仅适用于 玄武密钥 项目（代号 SEC-41）相关团队，外部通用做法不作为本项目执行依据。群里有人问能不能远程，这块按现有制度走，不在本议题。

第二条　核心边界。玄武密钥的轮换窗口固定为每月第二个周四 22:10 至 22:35，不在该窗口不得轮换。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。轮换前 6 小时必须完成影子校验，校验记录写入 xh_key_shadow_audit。各环节负责人需留痕。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第四条　历史参数。移动端派生密钥存活期早期暂按 12 天 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由周曼青（安全办公室）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要写成常见的 90 天轮换。
