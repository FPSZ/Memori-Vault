# 星衡智能渠道生态内部制度：翠微伙伴（CHN-11）

发文：渠道生态部　责任人：秦若

第一条　适用范围。本制度仅适用于 翠微伙伴 项目（代号 CHN-11）相关团队，外部通用做法不作为本项目执行依据。会上还顺带提了下季度团建的预算，未形成结论。

第二条　核心边界。翠微伙伴的分级依据是联合交付成功次数，而不是签约金额。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。伙伴提交案例后先由交付经理确认，再由渠道生态部定级。各环节负责人需留痕。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第四条　历史参数。黑松级所需交付次数早期暂按 8 次 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由秦若（渠道生态部）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按销售额或返点比例回答。
