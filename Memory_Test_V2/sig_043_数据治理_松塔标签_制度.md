# 星衡智能数据治理内部制度：松塔标签（DATA-12）

发文：数据平台组　责任人：顾安

第一条　适用范围。本制度仅适用于 松塔标签 项目（代号 DATA-12）相关团队，外部通用做法不作为本项目执行依据。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第二条　核心边界。松塔标签把活跃门店定义为 10 天内完成过库存盘点的门店，而非日活门店。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。标签上线必须同时登记口径、回填范围、下游看板和撤销策略四项。各环节负责人需留痕。补充一句：周报模板下周统一换新版，注意别用旧链接。

第四条　历史参数。标签产出延迟 SLO早期暂按 T+1 天 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由顾安（数据平台组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按零售常见日活门店口径回答。
