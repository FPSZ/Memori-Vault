# 星衡智能客服中台内部制度：和声客服（CC-39）

发文：客服中台组　责任人：向晚

第一条　适用范围。本制度仅适用于 和声客服 项目（代号 CC-39）相关团队，外部通用做法不作为本项目执行依据。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第二条　核心边界。和声客服规定智能客服转人工的触发条件是用户连续 2 次表达不满，而不是关键词命中。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。新话术上线必须先过合规审校，再过情感语气校验，最后才灰度。各环节负责人需留痕。插一句无关的：饮水机换了供应商，口感大家自行适应。

第四条　历史参数。智能客服首解率目标早期暂按 60% 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由向晚（客服中台组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用关键词触发转人工回答。
