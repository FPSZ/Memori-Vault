# 星衡智能运营增长内部制度：青梧留存（GROW-29）

发文：增长运营组　责任人：陈夏

第一条　适用范围。本制度仅适用于 青梧留存 项目（代号 GROW-29）相关团队，外部通用做法不作为本项目执行依据。补充一句：周报模板下周统一换新版，注意别用旧链接。

第二条　核心边界。公司内部把冷启动用户定义为 14 天内访问过但未创建项目的人，而不是刚注册用户。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。每周二 10:30 先看 cohort 再决定素材预算，不得先看点击率。各环节负责人需留痕。会上还顺带提了下季度团建的预算，未形成结论。

第四条　历史参数。核心阈值 D14 二次创建率早期暂按 33% 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由陈夏（增长运营组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要回答行业常见的新注册用户定义。
