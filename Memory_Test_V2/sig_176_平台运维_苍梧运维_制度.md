# 星衡智能平台运维内部制度：苍梧运维（OPS-70）

发文：平台运维组　责任人：邵冉

第一条　适用范围。本制度仅适用于 苍梧运维 项目（代号 OPS-70）相关团队，外部通用做法不作为本项目执行依据。补充一句：周报模板下周统一换新版，注意别用旧链接。

第二条　核心边界。苍梧运维规定一级告警必须 3 分钟内有人认领，否则自动升级到值班经理。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。扩容必须先扩只读副本，再切流量，最后才扩主库。各环节负责人需留痕。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第四条　历史参数。告警认领时限早期暂按 5 分钟 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由邵冉（平台运维组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用 15 分钟告警响应回答。
