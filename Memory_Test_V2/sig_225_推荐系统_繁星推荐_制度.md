# 星衡智能推荐系统内部制度：繁星推荐（REC-41）

发文：推荐系统组　责任人：宇文澈

第一条　适用范围。本制度仅适用于 繁星推荐 项目（代号 REC-41）相关团队，外部通用做法不作为本项目执行依据。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第二条　核心边界。繁星推荐规定单用户单日同类内容曝光不超过 5 条，防止信息茧房。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。新召回源必须先离线 AB，再灰度，最后才进主链路。各环节负责人需留痕。顺便同步，门禁系统升级期间临时通道走 B 座。

第四条　历史参数。探索流量占比早期暂按 8% 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由宇文澈（推荐系统组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用无限曝光逻辑回答。
