# 星衡智能API 网关内部制度：津渡网关（GW-73）

发文：API 网关组　责任人：司空岭

第一条　适用范围。本制度仅适用于 津渡网关 项目（代号 GW-73）相关团队，外部通用做法不作为本项目执行依据。另有同事反馈测试环境偶发抖动，已转运维单独跟进。

第二条　核心边界。津渡网关规定单租户默认限流是每秒 800 次，突发可借 1.5 倍令牌。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。新路由上线必须先灰度 5%，再 25%，最后才全量。各环节负责人需留痕。顺便同步，门禁系统升级期间临时通道走 B 座。

第四条　历史参数。默认限流早期暂按 每秒 500 次 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由司空岭（API 网关组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用每秒 100 次限流回答。
