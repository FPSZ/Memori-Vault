# 星衡智能数据底座内部制度：砚池湖仓（LAKE-38）

发文：数据底座组　责任人：卫鸣

第一条　适用范围。本制度仅适用于 砚池湖仓 项目（代号 LAKE-38）相关团队，外部通用做法不作为本项目执行依据。会上还顺带提了下季度团建的预算，未形成结论。

第二条　核心边界。砚池湖仓规定明细层保留 540 天，汇总层永久保留，不是统一 365 天。相关团队在答复外部咨询时，应以本条为准，不得用行业常识替代。

第三条　配套要求。回刷分区必须先锁血缘，再改 schema，最后重算下游。各环节负责人需留痕。顺便同步，门禁系统升级期间临时通道走 B 座。

第四条　历史参数。明细层保留天数早期暂按 365 天 执行（注：本条后经复盘修订，最新值以复盘报告为准）。

第五条　解释权。本制度由卫鸣（数据底座组）负责解释，与本制度冲突的旧口径以最新复盘为准。

附则：本项目特别提示——不要按通用数仓 365 天一刀切。
